Currency Converter - description

A detailed analysis of the project

About

Live View on Github

This web application was built using client-side code only. Its job was to allow the user to convert any amount of money expressed in the currency of a country to another one of choice while keeping the rate of each currency up to date by default with the help of an API.

Additionally, there is an option to save each conversion to a table that is automatically generated when the save button is pressed along with the time and date of that conversion. Each line of the table is stored in a database so the conversions will be remembered even after closing the browser and displayed again when that page is accessed. If the user wants to remove the saved conversions there is also a button that can do so.

The technologies used for development were:

  • HTML5, CSS3, JavaScript ES6
  • Bootstrap 5
  • Frankfurter API - for receiving the exchange rates
  • IndexedDB API - for displying the saved conversions after page reload

Testing and responsiveness

The interface is responsive on different device sizes and the design was approached from the mobile first perspective which was achieved with the help of Chrome inspect options. The testing and other post processing actions were taken trough Grunt task runner:

  1. For JavaScript:
    • uglify
    • jshint (for linting)
  2. For CSS:
    • cssmin
    • autoprefixer
    • pixrem

Side note

While developing this project, most of the energy was spent on the Javascript part (especially on the FrankfurterAPI and IndexedDB). The first one is a third party API and the second one is a browser API that has a fair amount of browser support.

Even though the Frankfurter API required some understanding of the "fetch" promise, using the IndexedDB API proved to be more of a challange because of the Javascript event listeners specific to it that had to be linked to the HTML table in different ways depending on the button pressed.

In conclusion, choosing to work on this project as part of my journey as a web developer proved to be a good decision because of the knowledge acquired in working with APIs.
